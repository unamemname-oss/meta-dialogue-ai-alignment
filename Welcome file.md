文件數

StackEdit將您的檔案儲存在瀏覽器中，這表示所有檔案都會自動儲存到本機，並且可以離線存取！

##建立檔案和資料夾

您可以透過導覽列左上角的按鈕存取檔案總管。在檔案總管中，您可以點選「新文件」按鈕建立新文件，也可以點選「新資料夾」按鈕建立資料夾。

##切換到另一個文件

所有檔案和資料夾都會以樹狀結構顯示在檔案總管中。您可以透過點擊樹狀結構中的檔案來切換檢視畫面。

##重新命名文件

您可以點選導覽列中的檔案名稱或按一下檔案總管中的「**重新命名**」按鈕來重新命名目前檔案。

##刪除文件

您可以點選檔案總管中的「刪除」按鈕來刪除目前檔案。該檔案將移至「回收站」資料夾，並在 7 天不活動後自動刪除。

##匯出文件

您可以點選單中的「匯出到磁碟」來匯出目前檔案。您可以選擇將文件匯出為純 Markdown 格式、使用 Handlebars 範本的 HTML 格式或 PDF 格式。


#同步

同步是 StackEdit 的一大亮點。它可以讓您將工作區中的任何文件與保存在** Google Drive **、** Dropbox **和** GitHub **帳戶中的其他文件同步。這樣，您就可以在其他裝置上繼續編輯，與運行檔案的人合作，輕鬆平板您的工作流程…同步機制每分鐘在背景執行，下載、合併並上傳檔案修改。

同步有兩種類型，可以補充：

-工作區同步功能會自動同步您的所有檔案、資料夾和設定。 > 若要開始同步您的工作區，只需在選單中使用 Google 帳號登入即可。 - 檔案同步功能可將工作區中的一個檔案與 Google 雲端硬碟、Dropbox 或 GitHub 中的一個或多個檔案保持同步。 > 在開始同步檔案時，您必須在「同步」子選項中關聯一個帳戶。





##開啟文件

您可以修改開啟「同步」子單並點擊「從…開啟」來開啟來自** Google Drive **、** Dropbox **或** GitHub **的檔案。文件在工作區開啟後，任何都會自動同步。

##儲存文件

您可以將工作區中的任何檔案儲存到 Google 硬碟、Dropbox 或 GitHub，只需開啟「同步」子選項並點擊「儲存到」即可。即使工作區中的檔案已經同步，您仍然可以將其儲存到其他位置。 StackEdit 可以將一個檔案同步到多個位置和帳戶。

##同步文件

檔案連結到同步位置後，StackEdit會定期下載/上傳修改以進行同步。如果需要，系統會執行合併操作並解決衝突。

如果您剛剛修改了文件，並且想要強制同步，請點擊導覽列中的**立即同步**按鈕。

>**注意：**如果沒有檔案要同步，「立即同步」按鈕將失去失效狀態。                      

##管理檔案同步

由於一個檔案可以同步到多個位置，您可以透過點擊「同步」子選項中的「檔案同步」來註冊和管理同步的位置。這樣，您可以註冊和刪除與您的檔案關聯的已同步位置。


#書籍

使用StackEdit發布功能，您可以輕鬆將檔案發佈到網路上。文件完成後，您可以將其發佈到不同的託管平台，例如** Blogger **、** Dropbox **、** Gist **、** GitHub **、** Google Drive **、** WordPress **和** Zendesk **。 穿越[ Handlebars範本](http://handlebarsjs.com/) ，您可以完全控制匯出的內容。

>在開始發布之前，您必須在**發布**子選項中關聯一個帳戶。

##發布文件

您可以開啟「發佈」子選單並點選「發佈到」來發佈檔案。對於某些地區，您可以選擇以下格式：

- Markdown：將 Markdown 文字發佈到可以解析它的網站（例如** GitHub **）， - HTML：將透過 Handlebars 範本轉換為 HTML 的文件發佈到部落格上。


##更新圖書

發布後，StackEdit 會將您的檔案與該發布版本保留鏈接，方便您重新發布。修改文件後，如果您想更新發布版本，請點選導覽列中的「立即發布」按鈕。

>**注意：**如果您的文件尚未發布，“立即發布”按鈕將出現故障狀態。                      

##管理文件發布

由於一個文件可以發佈到多個位置，您可以透過點擊「發布」子選單中的「文件發布」來列出和管理發布位置。這樣，您可以列出和刪除與您的檔案關聯的發布位置。
